# Contributing to Event Invitations Platform

Thank you for your interest in contributing! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Run `./setup.sh` (Mac/Linux) or `setup.bat` (Windows)
4. Create a new branch: `git checkout -b feature/your-feature`

## Development Workflow

1. Start dev server: `npm run dev`
2. Make your changes
3. Test thoroughly
4. Commit with clear message: `git commit -m "Add feature XYZ"`
5. Push to your fork: `git push origin feature/your-feature`
6. Create a Pull Request

## Code Standards

- Write TypeScript (no `any` types)
- Follow existing code style
- Add JSDoc comments for functions
- Keep components focused and reusable
- Use meaningful variable names

## Commit Messages

```
type(scope): subject

Where type is: feat, fix, docs, style, refactor, test, chore
Example: feat(templates): add new wedding template
```

## Pull Request Process

1. Update documentation if needed
2. Add tests if applicable
3. Ensure no linting errors: `npm run lint`
4. Get approval from maintainers
5. Squash and merge

## Areas We Need Help With

- 🐛 Bug fixes
- ✨ New templates
- 📚 Documentation improvements
- 🎨 UI/UX enhancements
- ⚡ Performance optimizations
- 🧪 Tests

## Questions?

Open an issue or discussion!
